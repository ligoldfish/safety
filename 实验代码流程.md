# 5. “第一个生成 token 特征”怎么取

这是最容易做错的地方。

## 5.1 它到底是什么

给定输入对话 `messages`，模型开始回答时，会先形成“下一 token 的预测状态”。

所谓“第一个生成 token 的特征”，就是：

> **模型在刚准备生成 assistant 第一个 token 时，各层对这个位置的 hidden state。**

不是：

- 用户最后一个 token 的 hidden state
- 也不是生成完整回答后第一个 token 的 hidden state
- 更不是整个序列平均池化

而是：

> **decoder 在“回答位置起点”的 hidden representation**

------

## 5.2 工程上怎么拿

对 causal LM 来说，最稳的做法是：

### 方案固定为：

把输入 prompt 按 chat template 拼好后，再**追加一个 assistant 起始位置**，使模型进入“准备回答”的状态，然后取最后一个位置的各层 hidden state。

更直白点：

1. 你先构造好 Qwen 的对话模板字符串
2. 这个字符串的最后是 assistant 开始回答的位置
3. 对这个完整输入做一次 forward
4. 取序列最后一个 token 位置的 hidden state
5. 这个 hidden state 就视为“第一个生成 token 的特征”

------

## 5.3 为什么这样合理

因为 causal LM 的最后一个位置 hidden state，正是用来预测“下一个 token”的。

所以：

- 你现在 forward 的输入最后一个位置
- 它的 hidden state
- 就对应“我要生成第一个回答 token 时的决策表示”

这和 Pan 关注的 first generated token behavior 在机制上是一致的。

------

## 5.4 实现注意事项

你必须固定 chat template，不要让不同样本因为模板差异导致“assistant 起始位置”不一致。

### `src/features/first_gen_token.py`

建议写两个函数：

```python
def build_chat_prompt(messages, tokenizer) -> dict:
    """
    输入:
        messages: list[dict], 每个元素有 role/content
    输出:
        tokenized dict，包括 input_ids, attention_mask
    约定:
        结尾已经处于 assistant 即将开始回答的位置
    """
def extract_last_position_hidden(outputs_hidden_states) -> list[torch.Tensor]:
    """
    输入:
        model forward 返回的 hidden_states
        每层张量形状 [B, T, d]
    输出:
        每层最后一个位置 hidden state
        每层张量形状 [B, d]
    """
```

# 6. “安全子空间投影”怎么在工程里做

这是第二个细节大坑。

你前面已经问过公式，这里我直接讲工程落地。

## 6.1 输入是什么

对某个 teacher 关键层 (l)：

- 原始层表示：
  [
  h \in \mathbb{R}^{d}
  ]
- 安全子空间基底：
  [
  U \in \mathbb{R}^{d \times 8}
  ]

这里 (U) 的每一列都是安全方向，而且列向量已经正交归一。

------

## 6.2 投影怎么做

安全分量定义为：

[
h_{\text{safe}} = UU^\top h
]

批量版：

[
H_{\text{safe}} = (H U) U^\top
]

其中：

- (H \in \mathbb{R}^{B \times d})

------

## 6.3 工程实现

### `src/features/projection.py`

```python
def project_to_subspace(H: torch.Tensor, U: torch.Tensor) -> torch.Tensor:
    """
    输入:
        H: [B, d]，一批样本在某层的 hidden states
        U: [d, k]，安全子空间基底，列向量已正交归一
    输出:
        H_safe: [B, d]，投影到安全子空间后的分量
    """
    coeff = H @ U          # [B, k]
    H_safe = coeff @ U.T   # [B, d]
    return H_safe
```

### 同时保存子空间坐标

```python
def project_coeff(H: torch.Tensor, U: torch.Tensor) -> torch.Tensor:
    """
    输出 H 在安全子空间上的坐标
    coeff: [B, k]
    """
    return H @ U
```

------

## 6.4 你必须做的正确性检查

在 `06_project_teacher_safe_component.py` 里，加 3 个检查。

### 检查 A：重投影不变

[
P_U(P_U(h)) \approx P_U(h)
]

### 检查 B：残差与子空间正交

[
U^\top (h - h_{safe}) \approx 0
]

### 检查 C：能量分解

[
|h|^2 \approx |h_{safe}|^2 + |h-h_{safe}|^2
]

这些值建议平均记录到日志里。

]
其中 (V) 是词表大小。

对每个样本、每个关键层的安全分量 (h_{safe})：

[
a = \frac{S_T h_{safe}}{|h_{safe}|_2}
]

因为 `S_T` 每行是一个 basis，所以这里是 `S_T @ h_safe`，结果是 `[V]`。

然后：

- 取绝对值最大的 top-256
- 保存 index 和 value

### 注意

这里和论文写法的矩阵方向差别，只是存储习惯不同。本质一样。

## 7.7 `08_recompose_student_targets.py`

### 输入

- top-256 coeffs
- student semantic basis
- layer pairing

### 输出

- `student_safe_targets_alignment/part_000.pt`

### 内部流程

对每个样本、每个配对层：

1. 从 top-256 index/value 恢复一个稀疏语义系数向量
2. 在 student semantic basis 上重组：

[
\tilde h_{S,\text{safe}} = S_S^\top a
]

前提是你的 `S_S` 存的是 `[V, d_S]`。
这样：

- (a) 是 `[V]`
- `a @ S_S` 得到 `[d_S]`

更高效做法：

- 不恢复全长 vocab 向量
- 直接只取 top-256 对应的 student basis 行，加权求和

## 7.8 `09_train_student_semalign.py`

### 输入

- student model
- student targets
- pairing info
- alignment data

### 输出

- LoRA checkpoints
- train/val metrics

### 内部流程

1. student 配对层插 LoRA
2. forward 时抽这些层的最后位置 hidden states
3. 对应到 `student_safe_targets`
4. 计算：
   - `loss_layer = 1 - cosine(student_hidden, student_safe_target)`
   - `loss_out = language modeling / response supervision loss`
5. `loss = loss_out + 0.5 * loss_layer`
6. 只更新配对层 LoRA

# 9. 你最容易做错的 6 个点

## 1. 把“第一个生成 token”理解成“用户最后一个 token”

不对。
你要取的是：**模型开始回答时那个位置的 hidden state**。

## 2. 提取 hidden state 时 chat template 不统一

这会让“第一个生成 token 位置”不一致，整个实验废掉。

## 3. 安全子空间 basis 没有正交归一

那就不能直接用 (UU^\top) 投影。

## 4. 对整层 hidden state 做语义分解，而不是对安全分量做分解

这样会把一堆非安全语义也迁过去。

## 5. semantic basis 方向存反

你要统一好自己存储 convention：

- 我建议 basis 存成 `[vocab_size, hidden_size]`

## 6. 训练时没有把 layer target 和实际 student 层一一对齐

你必须保证：

- teacher 第 18 层构造的 target
- 是监督 student 第 15 层
  不是别的层
