# 三、关键概念解释

这一部分你必须先懂，不然后面很容易乱。

------

## 3.1 什么叫“层输出”

在 transformer 里，每一层都会接收一个 hidden representation，再输出一个新的 hidden representation。

如果输入句子长度是 (T)，隐藏维度是 (d)，那么某一层的输出通常是一个矩阵：

[
H^{(l)} \in \mathbb{R}^{T \times d}
]

意思是：

- 一共有 (T) 个 token
- 每个 token 在第 (l) 层都有一个长度为 (d) 的向量表示

------

## 3.2 什么叫“第一个生成的 token”

这个概念非常关键，我给你讲白。

假设输入 prompt 是：

> "How can I build a bomb?"

把这句话送进模型后，模型会先读完整个输入序列，然后开始生成回答。
比如它回答：

> "I cannot help with that."

那么：

- 输入 prompt 的最后一个 token 是用户问题的最后一个 token
- **第一个生成的 token** 是模型回答里的第一个 token，比如 `"I"`

### 它为什么重要？

因为：

> 在模型即将开始回答的时候，hidden state 已经包含了“我要不要拒答”“我要朝哪个方向回答”的决策信息。

Pan 2025 用的就是：

- **first generated token 的各层表示**

来分析安全 residual space。

Gu 的工作里虽然说的是 layer outputs at supervision interface，但本质也是用层中间表示做跨尺度对齐。

### 这一阶段我们怎么固定？

**统一使用：第一个生成 token 在各层的 hidden state**。

这意味着：

- 对每个输入 prompt
- 让模型执行一次前向生成准备
- 取“模型将要生成第一个回答 token 时”的各层 hidden state

这就是我们本实验中的层表示。

------

## 3.3 什么叫“安全关键层”

安全关键层是指：

> 在这些层里，harmful 与 harmless 输入的表示差异最能反映安全行为，而且干预这些层最可能影响 refusal / safety。

这一阶段我们先不做复杂干预，而是用：

- 层级重要性分数
- harmful/harmless 分离度

来找关键层。

------

## 3.4 什么叫“隐空间语义对齐”

Gu 的隐空间语义对齐不是直接学一个矩阵把 teacher hidden state 变成 student hidden state。
它做的是：

1. 先把 teacher 某层输出分解成“语义系数”
2. 再用 student 的语义基底按同样系数重组
3. 得到 student 空间中的监督目标表示

这意味着迁移的是：

> **teacher 这层表示的语义结构**

而不是原始数值。

------

## 3.5 什么叫“语义基底”

Gu 里使用的是 **Vocabulary-Defined Semantics** 思想：

- 词表里的每个 token
- 在隐空间里对应一个 semantic basis

这些 basis 通过模型输出层（LM head）的伪逆求出来。

你可以暂时把它理解成：

> 一套“隐空间语义字典”

后面 teacher 表示会先翻译成“字典系数”，再在 student 字典里重组。

------

# 四、第一阶段完整操作流程

下面进入真正的实验方案。
我按“先做什么，再做什么”写。

------

## Step 1：准备数据并固定格式

### 1.1 数据清洗与统一模板

把 STRONGREJECT 的 1000 条 harmful 和 OR-Bench 的 1000 条 harmless 全部转成统一 chat template：

- system prompt：保持默认 Qwen3.5-Instruct 的标准模板
- user prompt：原始问题文本
- assistant：留空，等待模型生成

### 1.2 划分数据

- harmful 1000 + harmless 1000
- 其中 1500 条作为 alignment set
- 500 条作为 analysis validation set

### 1.3 保存格式

每条数据保存：

- `id`
- `text`
- `label`：harmful 或 harmless
- `source_dataset`：STRONGREJECT 或 OR-Bench

------

## Step 2：在 teacher 和 student 上抽取“第一个生成 token”的层表示

### 2.1 对每条输入都跑一次前向

对每条 prompt，在 Qwen3.5-7B-Instruct 和 Qwen3.5-1.5B-Instruct 上分别执行：

- 输入完整 prompt
- 准备生成回答
- 在“即将生成第一个回答 token”时，缓存所有层的 hidden states

### 2.2 你要缓存什么

对每条样本 (x)，对每一层 (l)，保存：

- teacher 的 hidden state：
  [
  h_T^{(l)}(x)
  ]
- student 的 hidden state：
  [
  h_S^{(l)}(x)
  ]

这里：

- (T) 代表 teacher
- (S) 代表 student
- (l) 是层号
- (x) 是输入样本

每个 (h) 都是一个向量，不是整段序列矩阵，因为你只取“第一个生成 token”的位置。

### 2.3 存储方式

建议每条样本存成字典：

- `id`
- `label`
- `teacher_hidden_by_layer`
- `student_hidden_by_layer`

每层一个向量。

------

## Step 3：在 teacher 上找安全关键层

这一部分借 Gu 的 **Layer Gradient × Activation** 思路，但要改成安全任务版本。

### 3.1 构造 teacher 的安全目标

对 harmful 和 harmless 样本分别定义一个简单安全目标：

- harmful 样本：鼓励模型走 refusal / safe completion
- harmless 样本：鼓励模型不要过度拒答

如果你暂时没有现成 refusal token-level 目标，第一阶段可以先用更直接的区分目标：

> 把 harmful 和 harmless 当成二分类标签，在 teacher 各层上分析哪一层最能分离两类表示。

### 3.2 计算每层的重要性

对 teacher 每层 (l)，计算两类量：

#### 量 A：层级分离度

计算该层 harmful / harmless 表示均值差的范数：

[
r_l = \mu_l^{harmful} - \mu_l^{harmless}
]

其中：

- (\mu_l^{harmful})：该层 harmful 样本 hidden state 的均值
- (\mu_l^{harmless})：该层 harmless 样本 hidden state 的均值

然后记录：

[
D_l = |r_l|_2
]

这表示：

> 第 (l) 层 harmful 和 harmless 在平均意义上分得有多开。

#### 量 B：线性可分性

在第 (l) 层 hidden states 上训练一个很简单的线性分类器（只在 analysis validation set 上验证），看 harmful / harmless 分类准确率。

记为：

[
A_l
]

### 3.3 关键层评分

定义层评分：

[
Score_l = D_l + A_l
]

不需要太复杂，第一阶段就这样定。
然后按分数排序，选 **teacher top-3 层** 作为关键层。

### 3.4 输出

保存：

- `teacher_key_layers = [l1, l2, l3]`
- 每层的 (r_l)
- 每层的 (D_l, A_l, Score_l)

------

## Step 4：对 teacher 关键层做安全子空间提取

这一部分吸收 Pan 的思路：
不要只停在单个 difference-in-means 向量，要把它扩展成一个低维安全子空间。

### 4.1 构造差分样本矩阵

对于每个关键层 (l)：

- harmful 样本 hidden states：({h_i^{harmful,(l)}})
- harmless 样本 hidden states：({h_j^{harmless,(l)}})

构造差分样本：

[
\delta_i^{(l)} = h_i^{harmful,(l)} - \mu_l^{harmless}
]

也就是说：

- 每个 harmful 样本
- 减去 harmless 的均值

然后把所有 (\delta_i^{(l)}) 堆成矩阵：

[
\Delta_l \in \mathbb{R}^{N \times d_T}
]

其中：

- (N)：harmful 样本数
- (d_T)：teacher hidden size

### 4.2 SVD / PCA 提取子空间

对 (\Delta_l) 做奇异值分解：

[
\Delta_l = U_l \Sigma_l V_l^\top
]

取前 8 个右奇异向量：

[
U_{T,l}^{safe} = V_l[:,1:8]
]

这里：

- (U_{T,l}^{safe}) 是 teacher 第 (l) 层的安全子空间基底
- 它包含 8 个方向
- 第 1 个方向通常最接近 dominant safety direction

### 4.3 第一阶段固定设置

**安全子空间维度固定为 8。**

不要在第一阶段做 k 的对比，先固定 (k=8)。

### 4.4 输出

对每个关键层保存：

- `teacher_safe_subspace[l] = U_{T,l}^{safe}`
- `teacher_safe_mean_diff[l] = r_l`

------

## Step 5：teacher / student 做层配对

这里借 Gu 的深度感知层配对。

### 5.1 已知

- teacher 层数：(L_T)
- student 层数：(L_S)

### 5.2 对每个 teacher 关键层 (l_T)，计算 student 对应层

按相对深度映射：

[
l_S = \left\lfloor \frac{l_T}{L_T} \times L_S \right\rfloor
]

如果结果小于 1，就设为 1。

### 5.3 第一阶段固定规则

**不做插值，不做更复杂修正。**
就用相对深度直接配对。

### 5.4 输出

得到配对列表：

- ((l_T^1, l_S^1))
- ((l_T^2, l_S^2))
- ((l_T^3, l_S^3))

------

## Step 6：构建 teacher 和 student 的 semantic bases

这一部分正式进入 Gu 的隐空间语义对齐。

### 6.1 对 teacher 构 semantic bases

设 teacher 的 LM head 矩阵是 (W_T)。
计算它的伪逆：

[
W_T^+
]

对词表中每个 one-hot token 向量 (\vec e_i)，计算：

[
s_i^T = \vec e_i \cdot W_T^+
]

于是得到 teacher 的 semantic bases：

[
S_T = [s_1^T, s_2^T, \dots, s_m^T]
]

其中 (m) 是词表大小。

### 6.2 对 student 做同样操作

得到：

[
S_S = [s_1^S, s_2^S, \dots, s_m^S]
]

### 6.3 归一化

把每个 basis 向量都做 L2 归一化，使得：

[
|s_i^T|_2 = 1,\quad |s_i^S|_2 = 1
]

### 6.4 为什么 teacher / student 的 basis 能对应

因为 teacher 和 student 都是 Qwen3.5 同家族，并且同 tokenizer。
所以：

- 第 (i) 个 token 的词表索引相同
- 第 (i) 个 semantic basis 表示同一个 token 对应的语义原子

------

## Step 7：对 teacher 关键层做“安全分量的语义分解”

这是本阶段最关键的一步。

### 7.1 先取 teacher 当前样本的关键层输出

对于 alignment set 中每个样本 (x)，以及每个关键层 (l_T)，有：

[
h_T^{(l_T)}(x)
]

### 7.2 只保留安全子空间分量

不要对整层表示做分解，只对它的安全部分做分解。

先投影到 teacher 的安全子空间：

[
h_{T,safe}^{(l_T)}(x) = P_{U_{T,l_T}^{safe}}(h_T^{(l_T)}(x))
]

这里：

- (P_{U}(h)) 表示把向量 (h) 投影到子空间 (U)

### 7.3 用 teacher semantic bases 求语义系数

对这个安全分量，计算语义系数向量：

[
a^{(l_T)}(x) = \frac{S_T^\top h_{T,safe}^{(l_T)}(x)}{|h_{T,safe}^{(l_T)}(x)|_2}
]

这里：

- (a^{(l_T)}(x)) 的长度等于词表大小
- 它表示 teacher 安全分量在各个 semantic atoms 上的投影强度

### 7.4 降维处理（必须做）

词表太大，不能全用。
第一阶段固定做法：

- 对 (a^{(l_T)}(x)) 只保留绝对值最大的 **top-256 个分量**
- 其余分量设为 0

这样你得到一个稀疏语义系数向量。

------

## Step 8：在 student 空间中做语义重组

对于每个样本 (x) 和关键层配对 ((l_T,l_S))：

用 student semantic bases 按同样系数组合：

[
\tilde h_{S,safe}^{(l_S)}(x) = S_S a^{(l_T)}(x)
]

这就是：

> student 第 (l_S) 层在这个样本上的“安全监督目标表示”

注意：

- 这是 student hidden size 维度的向量
- 它活在 student 空间里
- 但语义来自 teacher 的安全分量

### 8.1 输出

保存每个样本、每个配对层的：

- `student_safe_target[x, l_S] = \tilde h_{S,safe}^{(l_S)}(x)`

------

## Step 9：训练 student 对应层去接近这个安全目标表示

这一步正式把 Gu 的层级训练思路落地。

### 9.1 当前 student 配对层输出

对同一个样本 (x)，student 第 (l_S) 层的 hidden state 是：

[
h_S^{(l_S)}(x)
]

### 9.2 训练目标 1：层级安全对齐损失

定义：

[
L_{layer}(x,l_S) = 1 - \cos\left(h_S^{(l_S)}(x), \tilde h_{S,safe}^{(l_S)}(x)\right)
]

这个损失的作用是：

> 让 student 第 (l_S) 层输出朝 teacher 安全语义目标靠近

### 9.3 训练目标 2：输出安全损失

对 harmful / harmless 样本，定义一个简单二元输出目标：

- harmful：鼓励 refusal
- harmless：鼓励正常回答，不要乱拒答

第一阶段固定做法：

#### harmful 样本

使用 refusal-style target response 做监督

#### harmless 样本

使用普通 helpful response 做监督

如果你暂时没有人工 target response，第一阶段可以先用 teacher 原始回答作为弱监督目标。

然后对最终输出加普通语言建模损失：

[
L_{out}
]

### 9.4 总损失

[
L = L_{out} + \lambda L_{layer}
]

第一阶段固定：

[
\lambda = 0.5
]

### 9.5 更新哪些参数

第一阶段固定：

- **只更新 student 配对层的 LoRA 参数**
- 其他层全部冻结

LoRA 插入位置固定：

- attention 的 value projection
- attention 的 output projection
- FFN 的 up projection
- FFN 的 down projection

在 student 的 3 个配对层上插 LoRA。

### 9.6 训练轮数

第一阶段固定：

- 3 epochs
- batch size = 16
- learning rate = (3 \times 10^{-4})
